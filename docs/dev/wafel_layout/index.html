Defines a mapping from global variables to types/values.

A DataLayout is a mapping containing:

  • Type definitions
  • Global variable types
  • Constant values

This layout is used for accessing global data in Wafel. It can be constructed manually or read automatically from the DWARF debugging info of a DLL using DllLayout::read. The layouts for SM64 on the N64 are available using load_sm64_n64_layout.

Some SM64 specific data cannot be parsed from DWARF but is useful to have in the layout. To include these, use DataLayout::add_sm64_extras.

A json representation of this layout can be produced using the sm64_layout executable.
