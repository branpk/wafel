pub enum DataType {
    Void,
    Int(IntType),
    Float(FloatType),
    Pointer {
        base: DataTypeRef,
        stride: Option<usize>,
    },
    Array {
        base: DataTypeRef,
        length: Option<usize>,
        stride: usize,
    },
    Struct {
        fields: IndexMap<String, Field>,
    },
    Union {
        fields: IndexMap<String, Field>,
    },
    Name(TypeName),
}
Expand description

A representation of a C data type.

Variants§

§

Void

Void, typically used as a pointer target or function return type.

§

Int(IntType)

An integer type.

§

Float(FloatType)

A float type.

§

Pointer

Fields

§base: DataTypeRef

The type being pointed to.

§stride: Option<usize>

The size of the type being pointed to, if known.

This is used when applying a subscript to a pointer, e.g. p[3]. It is otherwise not necessary.

A pointer type.

§

Array

Fields

§base: DataTypeRef

The element type.

§length: Option<usize>

The length of the array, if known.

§stride: usize

The size of the element type.

This is used when indexing into the array.

An array type, optionally with a length.

§

Struct

Fields

§fields: IndexMap<String, Field>

The fields contained in the struct.

Anonymous fields should be given a name on construction, typically __anon.

A struct type.

§

Union

Fields

§fields: IndexMap<String, Field>

The fields contained in the union.

Anonymous fields should be given a name on construction, typically __anon.

A union type.

§

Name(TypeName)

A symbolic reference to a type definition, e.g. struct Foo.
